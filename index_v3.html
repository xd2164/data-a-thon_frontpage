Welcome and Overview

Welcome! Thank you for joining the AI-Enabled Career Navigation Datathon, hosted by the Gates Foundation and the Schultz Family Foundation.

We're gathering at a pivotal moment for career navigation. AI is reshaping the workforce—changing skill requirements, career trajectories, and entry-level opportunity. As this accelerates, our tools must evolve beyond role and wage navigation to show where opportunity actually develops.

This datathon brings leaders together to build, test, and experiment with new career navigation approaches. Using the American Opportunity Index as our data foundation, we'll prototype next-generation tools that help young adults and workers make more informed, equitable decisions.

Our goal: transform data into actionable infrastructure. Together, we'll advance AI-enabled navigation tools grounded in real mobility outcomes, shaped by practitioner insight, and informed by young people's lived experience.

Over two days, you'll collaborate with peers, youth participants, and technical advisors to explore how data, technology, and design improve long-term outcomes. We're excited to build alongside you.

The Challenge Prompt

Purpose and Opportunity

Large language models and generative AI are reshaping how learners and workers navigate the labor market. Without transparent, equitable data foundations, these tools risk reinforcing bias and misinformation. The American Opportunity Index (AOI)—covering 1,700 companies, 12 million workers, and 923 O*NET occupation codes—enables AI navigation tools grounded in real mobility outcomes, not unverified internet data. This Gates–Schultz Datathon prototypes next-generation tools that use AOI data responsibly to surface mobility-friendly pathways for young people, especially those in under-resourced communities.

Challenge Prompts & Use Cases

Each organization knows its users, contexts, and constraints best. This Datathon isn't about one-size-fits-all solutions—it's about creating compelling, user-centered tools that bring the American Opportunity Index to life. Start with your real users—young people, learners, workers, or advisors—and design AI-enabled experiences that translate data into meaningful, trustworthy, actionable insights. Success comes from clear use cases, thoughtful data integration, and strong narratives showing how your tool improves career navigation and economic mobility. Consider these use cases:

Use Case 1: Mobility Matchmaker

Career mobility is often discussed but rarely operationalized. Job seekers see titles and wages but lack visibility into whether employers or pathways support advancement over time. This use case explores signals—credentials, skills progression, promotion patterns, retention, local labor market dynamics—that characterize "mobility-friendly" pathways. Explore how different mobility definitions shape recommendations and surface promising opportunities not obvious through traditional job search tools.

Use Case 2: Explainable Career Paths

Many AI career tools generate recommendations without explaining rationale, limiting trust and agency. This use case examines how LLMs and structured outcomes data can articulate why certain transitions, credentials, or pathways lead to stronger outcomes. Focus on explanation over prediction—explore narrative framing, comparative reasoning, and transparency for different audiences (learners, families, advisors).

Use Case 3: Bias Audit Toolkit

AI systems can unintentionally reproduce inequities when outcomes differ across demographic or geographic groups. This use case focuses on analytical approaches to examine equity in career recommendations. Explore detecting disparities, defining fairness metrics for career navigation, and visualizing differences in access or outcomes. Understand where bias emerges, how to measure it, and what reporting mechanisms support accountability.

Use Case 4: Youth-Facing Interface

Career navigation tools often assume high prior knowledge, digital fluency, and institutional familiarity—assumptions that fail for many first-generation or early-career users. This use case explores translating data-driven insights into accessible, supportive guidance. Examine how language, framing, pacing, and representation influence comprehension and trust. Focus on sense-making and user experience, not technical implementation.

Use Case 5: Data Gap Discovery Toolkit

Data availability and quality limitations constrain what AI navigation systems can support. This use case invites identifying missing or underdeveloped data that most limit current approaches—informal learning, non-linear pathways, lived experience, long-term mobility outcomes. Explore how gaps affect recommendations and consider strategies for addressing them: data partnerships, new collection methods, or governance models.

The Data

Key Links: Data Dictionary

The American Opportunity Index (AOI) dataset provides an outcomes-based view of worker mobility across the largest U.S. employers, assessing companies and company-occupation pairs across 3 dimensions of job quality. Built from longitudinal analysis of 12 million worker career histories, combined with job postings and wage data, covering 1,763 large U.S. employers, 563 occupational clusters, and over 55,000 employer-occupation pairs.

The dataset provides stakeholders insights on employers and jobs that best match their priorities. Job quality isn't uniform: the same employer may offer strong opportunity in some occupations and limited opportunity in others. Assessing at the company-occupation level brings actionability to these variances.

We evaluate jobs and employers across core outcomes that matter most to workers (wages, stability/retention, promotion/mobility). These metrics combine into three distinct archetypes: Early Career jobs are accessible to entry-level workers and lead to promotion and advancement; Growth jobs enable upward mobility within and beyond the firm, particularly in organizations that promote from within; Stability jobs offer strong pay and long-term retention. Employers and employer-occupation pairs are rated using Platinum (top 20%) and Gold (next 20%) designations, showing both overall performance and role-specific strengths.

Core Metrics & Data Details

Core Metrics

  • Wages: Salary and wages for an occupation at a company as measured between 2019 and 2024, normalized to 2024 dollars, based on Glassdoor data.
  • Retention: Likelihood that a worker remains at the company for at least three years, based on observed job starts between 2017 and 2021.
  • Internal Promotion: Of workers who started in a given job between 2015 and 2019, this metric measures the share of workers who are in a higher paying role at the same employer within 5 years (i.e., 2020 thru 2024).
  • External Promotion (Advancement Beyond): Of workers who started in a given job between 2015 and 2019, this metric measures the share of workers who find themselves in a higher paying role at a different employer within 5 years (i.e., 2020 thru 2024).
  • Entry Level: Share of new hires with fewer than three years of experience, based on observed job starts between 2019 and 2024.
  • Leaders From Within: Share of top earners who rose into leadership roles internally. This is only metric that is calculated at the company level, as it reflects firm-wide leadership pathways rather than occupation-specific outcomes.

Three Job Quality Archetypes

Early Career Jobs

Composite of internal promotion and external promotion, filtered for company-occupation pairs with over 20% of hires at the entry level.

Stability Jobs

Composite of wages and retention.

Growth Jobs

Composite of internal promotion and external promotion, filtered for companies with strong performance on Leaders from Within.

Additional Data Features

  • Occupational Benchmarks: Typical wages, retention, and mobility for jobs, segmented by experience level and metro size.
  • Premium Skills: Skills that deliver the strongest wage premiums within each occupation.
  • Next-Step Occupations: Common next-step occupations that workers move into from each occupation.
  • Hiring Signals: Company-occupation-metro level data to help navigate users to firms who may be hiring for their skillsets.
  • AI Impact Indicators: Forward-looking indicators of how artificial intelligence is likely to reshape access and expertise within occupations.

Submission Details

How to Submit Your Project

All teams are required to submit their code and project materials via GitHub. Please follow the instructions below to ensure your submission is complete and properly formatted.

Submission Requirements

Submission Requirements

  • GitHub Repository: All code, documentation, and project files must be submitted to a GitHub repository
  • Repository Access: Ensure the repository is set to public or provide access to the judging team
  • README File: Include a comprehensive README.md file with project description, setup instructions, and usage guide
  • Code Organization: Maintain a clean, organized codebase with clear file structure and comments
  • Documentation: Include any additional documentation, data sources, or resources used in your project
Step-by-Step Submission Process

Step 1: Create GitHub Repository

Create a new repository on GitHub for your team's project. Name it clearly to identify your team and project (e.g., "team-name-aoi-datathon-2024").

  • Go to github.com
  • Click "New repository"
  • Choose a descriptive name
  • Set repository visibility (public or private with access granted)
  • Initialize with a README.md file

Step 2: Organize Your Code

Structure your repository with clear organization. Include separate directories for code, data, documentation, and presentation materials.

  • Create folders: /src, /docs, /data, /presentation
  • Add clear comments to your code
  • Include requirements.txt or package.json for dependencies
  • Document your API endpoints and data sources

Step 3: Create README.md

Your README should provide a clear overview of your project, including purpose, features, setup instructions, and how to run the application.

  • Project title and description
  • Team members and roles
  • Installation and setup instructions
  • Usage examples and API documentation
  • Link to live demo (if applicable)
  • Acknowledgments and data sources

Step 4: Push Your Code

Upload all your project files to the repository. Make sure to commit regularly with clear commit messages.

  • Initialize git in your project directory
  • Add all project files: git add .
  • Commit with descriptive message: git commit -m "Initial project submission"
  • Push to GitHub: git push origin main

Step 5: Submit Repository Link

Submit your GitHub repository URL through the official submission portal or email to the organizing team by the deadline.

  • Copy your repository URL (e.g., https://github.com/xd2164/data_a_thon)
  • Submit via the provided submission form
  • Include your team name and project title
  • Deadline: Check agenda for submission deadline
