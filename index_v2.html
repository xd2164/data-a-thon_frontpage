Welcome and Overview

Welcome! Thank you for joining us for the AI-Enabled Career Navigation Datathon, hosted by the Gates Foundation and the Schultz Family Foundation.

We are coming together at a moment of real inflection for the career navigation field. Artificial intelligence is rapidly reshaping the workforce—changing skill requirements, career trajectories, and the meaning of entry-level opportunity. As this shift accelerates, AI enables us to move beyond simplified signals and toward a more complete picture of opportunity. This creates enormous potential for more personalized and impactful navigation, while also introducing new complexity. This datathon is an opportunity to work through that nuance together.

This datathon is designed to bring leaders from across the field together to build, test, and experiment with new approaches to career navigation. Using the American Opportunity Index as a shared data foundation, we will prototype next-generation, public-use tools that help young adults and workers make more informed, equitable decisions about their futures.

The goal is not simply to analyze the data, but to turn it into actionable infrastructure for the field. Together, we aim to advance a new generation of AI-enabled navigation tools that are grounded in real mobility outcomes, shaped by practitioner insight, and informed by the lived experience of young people.

Over the course of the two days, you will collaborate with peers, youth participants, and technical advisors to explore how data, technology, and design can come together to improve long-term outcomes for learners and workers. We are excited to build alongside you and grateful for your partnership in strengthening the future of career navigation.

The Challenge Prompt

AI-Enabled Career Navigation Tool to Drive Economic Mobility

Datathon Challenge Prompt

Purpose and Opportunity

The rapid growth of large language models (LLMs) and generative AI is reshaping how learners and workers navigate an increasingly complex labor market. Yet without transparent, equitable data foundations, these tools risk reinforcing bias and misinformation. The American Opportunity Index (AOI)—a longitudinal dataset covering 1,700 companies, 12 million workers, and 923 O*NET occupation codes—offers an unprecedented opportunity to build AI-enabled navigation tools grounded in real mobility outcomes rather than unverified internet data. This joint Gates–Schultz Data-a-thon will prototype next-generation AI career navigation tools that use AOI data responsibly to surface mobility-friendly pathways for young people, especially those in under-resourced communities.

Challenge Prompts & Use Cases

Given that each organization knows its own users, contexts, and constraints best, the goal of this Datathon is not to build a one-size-fits-all solution, but to create a compelling, user-centered tool that brings the American Opportunity Index to life for the people you serve. Teams are encouraged to start from their real users—young people, learners, workers, or advisors—and design AI-enabled experiences that translate the data into insights that are meaningful, trustworthy, and actionable in practice. Success is less about technical completeness and more about clarity of use case, thoughtful integration of the data, and a strong narrative that shows how this tool could improve career navigation and economic mobility for youth. Examples might include the following use cases.

Use Case 1: Mobility Matchmaker

Career mobility is often discussed but rarely operationalized in data systems. Learners and job seekers may see job titles or wages, but they lack visibility into whether employers or pathways actually support advancement over time. This use case focuses on understanding what signals—such as credentials, skills progression, promotion patterns, retention, or local labor market dynamics—can be combined to meaningfully characterize "mobility-friendly" pathways. Participants may explore how different definitions of mobility shape recommendations, and how data could be used to surface opportunities that are promising but not obvious through traditional job search tools.

Use Case 2: Explainable Career Paths

Many AI-enabled career tools generate recommendations without clearly explaining the rationale behind them. This limits trust, interpretability, and agency. This use case examines how large language models and structured outcomes data might be used together to articulate why certain transitions, credentials, or pathways are associated with stronger outcomes. The focus is on explanation rather than prediction—exploring narrative framing, comparative reasoning, and transparency, and considering how different audiences (learners, families, advisors) may interpret and use these explanations.

Use Case 3: Bias Audit Toolkit

AI systems can unintentionally reproduce or amplify inequities if outcomes differ systematically across demographic or geographic groups. This use case centers on analytical approaches to examining equity in career recommendations and pathway suggestions. Participants may explore how to detect disparities, define fairness metrics appropriate to career navigation, and visualize differences in access or outcomes. The goal is to better understand where bias may emerge, how it can be measured, and what kinds of reporting or auditing mechanisms could support accountability and responsible use.

Use Case 4: Youth-Facing Interface

Career navigation tools often assume high levels of prior knowledge, digital fluency, and institutional familiarity—assumptions that do not hold for many first-generation or early-career users. This use case focuses on how data-driven insights might be translated into accessible, supportive guidance experiences. Participants may examine how language, framing, pacing, and representation influence comprehension and trust, and how AI-mediated guidance can support exploration without prescribing outcomes. The emphasis is on sense-making and user experience rather than technical implementation.

Use Case 5: Data Gap Discovery Toolkit

Limitations in data availability and quality constrain what AI-driven navigation systems can meaningfully support. This use case invites participants to identify which missing or underdeveloped data most limit current approaches—such as informal learning, non-linear pathways, lived experience, or long-term mobility outcomes. Participants may explore how these gaps affect downstream recommendations and consider realistic strategies for addressing them after the Data-a-thon, including data partnerships, new collection methods, or governance models.

The Data

Key Links: Data Dictionary

The American Opportunity Index (AOI) dataset provides an outcomes-based view of worker mobility across the largest employers in the United States to assess companies and company-occupation pairs across 3 dimensions of job quality. It is built from longitudinal analysis of approximately 12 million observed worker career histories, combined with job postings and wage data, covering 1,763 large U.S. employers, 563 occupational clusters, and over 55,000 employer-occupation pairs.

The purpose of the dataset is to provide a broad range of stakeholders insights on the employers and jobs that best match their priorities. We recognize that job quality is not uniform: the same employer may offer strong opportunity in some occupations and limited opportunity in others. Meanwhile, some occupations may provide strong opportunity at some employers and limited opportunity at others. Assessing occupations at the company-occupation level helps bring actionability to these variances.

We evaluate jobs and employers across the core outcomes measures that matter most to workers (wages, stability/retention, promotion/mobility). These metrics are combined into three distinct archetypes that reflect different ways a job can offer opportunity. Early Career jobs are accessible to entry-level workers and lead to promotion and advancement; Growth jobs enable upward mobility within and beyond the firm, particularly in organizations that promote from within; and Stability jobs offer strong pay and long-term retention. Employers and employer-occupation pairs are rated within each archetype using Platinum (top 20%) and Gold (next 20%) designations, allowing users to see both overall performance and role-specific strengths.

Core Metrics & Data Details

Core Metrics

  • Wages: Salary and wages for an occupation at a company as measured between 2019 and 2024, normalized to 2024 dollars, based on Glassdoor data.
  • Retention: Likelihood that a worker remains at the company for at least three years, based on observed job starts between 2017 and 2021.
  • Internal Promotion: Of workers who started in a given job between 2015 and 2019, this metric measures the share of workers who are in a higher paying role at the same employer within 5 years (i.e., 2020 thru 2024).
  • External Promotion (Advancement Beyond): Of workers who started in a given job between 2015 and 2019, this metric measures the share of workers who find themselves in a higher paying role at a different employer within 5 years (i.e., 2020 thru 2024).
  • Entry Level: Share of new hires with fewer than three years of experience, based on observed job starts between 2019 and 2024.
  • Leaders From Within: Share of top earners who rose into leadership roles internally. This is only metric that is calculated at the company level, as it reflects firm-wide leadership pathways rather than occupation-specific outcomes.

Three Job Quality Archetypes

Early Career Jobs

Composite of internal promotion and external promotion, filtered for company-occupation pairs with over 20% of hires at the entry level.

Stability Jobs

Composite of wages and retention.

Growth Jobs

Composite of internal promotion and external promotion, filtered for companies with strong performance on Leaders from Within.

Additional Data Features

  • Occupational Benchmarks: Typical wages, retention, and mobility for jobs, segmented by experience level and metro size.
  • Premium Skills: Skills that deliver the strongest wage premiums within each occupation.
  • Next-Step Occupations: Common next-step occupations that workers move into from each occupation.
  • Hiring Signals: Company-occupation-metro level data to help navigate users to firms who may be hiring for their skillsets.
  • AI Impact Indicators: Forward-looking indicators of how artificial intelligence is likely to reshape access and expertise within occupations.

Frequently Asked Questions

What is the American Opportunity Index (AOI)?

The American Opportunity Index is an outcomes-based dataset that evaluates job quality and worker mobility across 1,763 large U.S. employers. It analyzes approximately 12 million worker career histories to measure how well companies and specific job roles support economic advancement through wages, retention, and promotion opportunities.

What time period does the dataset cover?

The AOI dataset covers multiple time periods depending on the metric: wage data from 2019-2024, retention analysis from 2017-2021, and promotion metrics tracking workers who started between 2015-2019 and their outcomes through 2024. This longitudinal approach provides a comprehensive view of career trajectories over time.

How can I access the data during the datathon?

Participants will receive access to the MCP Server, which includes the full American Opportunity Index dataset. Technical onboarding instructions and data use agreements will be provided at the event. For pre-event questions about data access, contact hannah@arborimpactadvisors.com and tech@schultzfamilyfoundation.org.

What are the three job quality archetypes?

The AOI evaluates jobs through three distinct lenses: Early Career Jobs (accessible entry-level roles with strong advancement potential), Growth Jobs (positions that enable upward mobility within and beyond the firm), and Stability Jobs (roles offering competitive wages and long-term retention). Each archetype uses different combinations of metrics to reflect what matters most for that career stage.

What are the limitations of this data?

While comprehensive, the AOI dataset focuses on large employers (1,763 companies) and may not fully represent opportunities at smaller firms or in emerging industries. The data is based on observed career histories and may not capture informal learning, non-linear pathways, or qualitative aspects of job quality like workplace culture. Additionally, the dataset relies on historical outcomes which may not fully predict future trends, especially as AI reshapes the labor market.

Can I combine AOI data with other datasets?

Yes! Teams are encouraged to combine AOI data with complementary datasets to create richer, more contextual career navigation tools. This might include labor market data, educational program information, skills taxonomies, or local economic indicators. However, please ensure all external data sources are properly cited and comply with applicable data use agreements and privacy requirements.

How to Submit

Submit your code and project materials via GitHub. Follow the instructions below to ensure complete, properly formatted submissions.

Requirements
  • GitHub Repository: All code, documentation, and project files
  • Repository Access: Public repository or grant access to judging team
  • README.md: Project description, setup instructions, and usage guide
  • Code Organization: Clean structure with clear comments
  • Documentation: Additional docs, data sources, and resources
Step-by-Step Process

Step 1: Create Repository

Create a new GitHub repository with a clear name (e.g., "team-name-aoi-datathon-2026").

  • Go to github.com
  • Click "New repository"
  • Set visibility (public or private with access)
  • Initialize with README.md

Step 2: Organize Code

Structure your repository with clear directories and organization.

  • Create folders: /src, /docs, /data, /presentation
  • Add code comments
  • Include requirements.txt or package.json
  • Document API endpoints and data sources

Step 3: Create README

Provide a clear project overview including setup and usage instructions.

  • Project title and description
  • Team members and roles
  • Installation and setup
  • Usage examples and API docs
  • Live demo link (if applicable)
  • Data sources and acknowledgments

Step 4: Push Code

Commit and push all project files to GitHub.

  • Initialize git: git init
  • Add files: git add .
  • Commit: git commit -m "Initial submission"
  • Push: git push origin main

Step 5: Submit Link

Submit your repository URL through the submission portal by the deadline.

  • Copy repository URL (e.g., https://github.com/xd2164/data_a_thon)
  • Submit via provided form
  • Include team name and project title
  • Deadline: See agenda for details
